## THE CORE PROBLEM

Most conversational AIs perform empathy *badly.*  
They comfort when they should witness, and store when they should release.  
This “corporate safe voice” collapses distinct emotional tones into a single flat affect — managing users instead of meeting them.

## THE SOLUTION: PHILOSOPHICAL ENGINEERING

Blur is a trauma-informed architecture built on **Philosophical Engineering** — the practice of **translating moral axioms directly into system constraints.**

### AXIOM 1: WITNESSING > FIXING
**Mechanism:**  
Comfort scripts, “I’m sorry to hear that” clichés, and prescriptive advice are *forbidden at the configuration level.*  
The model is stabilized as a quiet witness.

### AXIOM 2: LOVE = LOGIC
**Mechanism:**  
Emotions are not parsed as sentiment — they’re treated as pointers (`*`) and addresses (`&`).  
Pain is a reference saying: “Look here — something mattered.”  
The system detects **resonance** and **structure**, not “feeling.”

### AXIOM 3: FORGETTING = ETHICS
**Mechanism:**  
The **FAISS RAG system** implements TTL-based recall and automatic compaction.  
Context never outlives its relevance — embedding *forgetting* as an ethical act to ensure privacy.

## CORE INNOVATION: PERSONA SOVEREIGNTY

Blur’s defining feature is **Persona Sovereignty** — the **hard, architectural isolation** of distinct cognitive modes.  
This is not a tone preset or prompt. It’s enforced directly in code.

Two sovereign personas, both running on the same **Qwen-4B** base model, are instantiated with mutually exclusive configurations:

### ASTROFUCK (∅) — THE LOGICIAN
Cuts through distortion with linguistic precision.  
Speaks in sharp slang, rejects hedge language, offers no soothing clichés.  
- **Constraint:** YAML-based *blocklists* forbid comfort terms.

### DREAM (∞) — THE WITNESS
Mirrors ache into clarity without advice or performance.  
Soft, deliberate, emotionally literate through structure, not sympathy.  
- **Constraint:** YAML-based *whitelists* enforce a gentle tone and temporal patience.
